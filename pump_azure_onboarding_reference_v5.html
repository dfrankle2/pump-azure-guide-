Quick identification cheat sheet
Ask the customer how they pay their Azure bill before anything else.
PAYGO
Pay-as-you-go / customer MCA
• Plan = Azure Plan
• Paid via credit card
• Billed directly by Microsoft
• "MCA" in portal but no enterprise terms
• No Microsoft account team
MCA-E
Microsoft Customer Agreement – Enterprise
• Plan = Azure Plan
• Paid via invoice (NET 30) from Microsoft
• Confirm by pulling an invoice — biller = Microsoft
• Does not necessarily mean they have a Microsoft account team
CSP
Cloud Solution Provider
• Invoice from a partner, not Microsoft
• Billing owned by a reseller/distributor
• Ask: "Who do you pay your Azure bill to?"
EA
Enterprise Agreement
• Portal shows Enterprise Agreement + enrollment number
• Means: they have a Microsoft account team
• Does not mean: annual spend commitment (separate concept)
• Direct EA → can transfer to CSP directly
• Indirect EA → must go MCA-E first, then CSP
• No new EAs sold — existing ones are transferable
Misc / Other
Sponsorship Microsoft for Startups
Customer has Azure credits through a Microsoft startup program. May have little or no real billing yet — credits cover spend.
Cannot work with Pump until credits expire. Track expiry date and follow up.
MAC = EDP
Microsoft Azure Consumption Commitment / Enterprise Discount Program. Customer has committed to a minimum Azure spend in exchange for discounts. Special commercial arrangement with Microsoft.
Cannot work with Pump until MAC expires. Get the expiry date — put a reminder to follow up.
ISV Independent Software Vendor
Typically has a special relationship with Microsoft — may include co-sell agreements, ISV discounts, marketplace benefits, or dedicated Microsoft resources.
Complexity varies. Confirm what benefits they have before starting — some may not carry over to CSP. Escalate to Ingram.

Adding an Identity via Entra ID & Granting Billing Account Owner Role
The user who accepts the CSP transfer request must be the Billing Account Owner on the MCA-E billing account. If the right person doesn't have that role, the transfer link will fail or be unacceptable.
Considerations & tips and tricks
Same person rule: The person who accepts the MCA-E proposal link must be the same person who accepts the CSP transfer request. Do not split these between different users — it will fail.
Tenant requirement: To accept an MCA-E, the user must belong to the correct tenant — the one that holds the billing account. A guest/external account cannot hold the Billing Account Owner role. If the right person is in a different tenant, a member account must be created for them in the billing tenant first.
Step 1 — Add the identity in Entra ID (if the user doesn't already exist)
1
Go to Entra ID
Navigate to entra.microsoft.com or search "Microsoft Entra ID" in the Azure portal.
2
Identity → Users → All Users
3
+ New user → Create new user
Or Invite external user if they're outside the tenant — but note guest accounts cannot hold billing roles for transfer purposes (see considerations above).
4
Fill in details
• Display name
• User principal name (e.g., admin@contoso.onmicrosoft.com)
• Set a temporary password
5
Click Create
Step 2 — Assign Billing Account Owner role
1
Azure Portal → Cost Management + Billing
2
Select the correct Billing Account (the MCA-E one)
Under Billing scope, select the MCA-E billing account.
3
Left menu → Access control (IAM)
4
+ Add → Add role assignment
Select role: Billing account owner. Search for and select the user. Click Save.
Key rules
Must be in the same tenant
The Billing Account Owner must be a member user in the tenant that holds the MCA-E billing account. Guest/B2B accounts cannot hold billing roles for transfer purposes.
Global Admin ≠ Billing Owner
Being an Entra Global Admin does not automatically make someone a Billing Account Owner — they are separate roles that must be assigned independently.
Multiple owners allowed
There can be multiple Billing Account Owners, but only one "true" owner via the ownership transfer flow.
Email access required
The billing owner must have access to their email — the transfer link is sent via email and expires.
Quick checklist before initiating the transfer
☐  Correct billing account identified (the MCA-E one, not a PAYGO profile)
☐  User exists as a member (not guest) in that tenant
☐  User has Billing Account Owner role under Cost Management + Billing → Access control (IAM)
☐  User has access to their email (transfer link is sent via email)
☐  Confirmed this is the same person who will accept both the MCA-E and the CSP transfer
Export quotas
Why export quotas? Quotas are reset when a subscription moves between billing types or tenants. Export before the transfer so you have a record to request restoration if needed.
Method 1 — Azure Portal (quick, visual, per-region)
1
Go to portal.azure.com
2

3
Select "Compute" (or the service type needed)
Options include Storage, Networking, etc.
4
Filter by Subscription and Region
5
Click Export to CSV in the top command bar
Currency changes & reservations
If the billing currency changes (e.g., GBP → EUR, AUD → USD), all existing reservations must be repurchased — they cannot be migrated across currencies. Identify any active reservations before the transfer and flag this to the customer.
